=== Datenbank-Workloads



==== Individuelle Einsparungen für Oracle-Hosts im Amazon Elastic Block Store

NetApp Workload Factory for Databases ermöglicht es Ihnen, die Einsparungen für Oracle-Hosts auf Amazon Elastic Block Store (EBS) individuell anzupassen. Sie können verschiedene Felder wie die Snapshot-Frequenz, die Anzahl der geklonten Kopien und die Volume-Details anpassen, um zu sehen, wie sie sich auf Ihre Einsparungen auswirken.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html["Entdecken Sie Einsparungen für Oracle-Hosts auf EBS"]

* link:https://docs.netapp.com/us-en/workload-databases/create-database-server.html["Microsoft SQL Server-Hosts bereitstellen"]
* link:https://docs.netapp.com/us-en/workload-databases/create-postgresql-server.html["PostgreSQL-Hosts bereitstellen"]

=== Amazon FSX für NetApp ONTAP



==== Blockgeräteklone

Workload Factory bietet Klone für Ihre Blockgeräte, sodass Sie schnell beschreibbare und effiziente Kopien Ihrer Daten bereitstellen und gleichzeitig Kapazität, Zugriff und Lebenszyklus kontrollieren können.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Blockspeicher in NetApp Workload Factory verwalten"]



==== EC2-Tags auf Blockgeräten

Die Blockgerätetabelle enthält nun eine Spalte mit den Tags der EC2-Hosts, die die jeweiligen Blockgeräte verwenden. Dies hilft Ihnen zu erkennen, welcher Host welches Blockgerät während der Fehlersuche oder bei Audits verwendet.

Um Tags auf jedem Blockgerät zu erstellen, stellen Sie eine EC2-Instanz mithilfe unseres bereitgestellten Benutzerskripts bereit.

link:https://github.com/NetApp/FSx-ONTAP-samples-scripts/tree/main/Management-Utilities/ec2-user-data-iscsi-create-and-mount["EC2-Benutzerdatenskript zum Zuweisen von Tags zu Blockgeräten"]



==== Kapazitätstrends für FSx für ONTAP-Dateisysteme

Erfahren Sie mehr über SSD-Kapazitätstrends direkt in der Übersicht Ihrer FSx for ONTAP-Dateisysteme. Die neue Karte für Kapazitätsmanagement zeigt die Nutzungstrends des Vormonats an. Diese Analyse vergleicht die tatsächliche und die zugewiesene Kapazität und ermöglicht so eine einfache Beurteilung der Ressourcennutzung.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-capacity-automatically.html["Kapazität automatisch in NetApp Workload Factory verwalten"]



==== PDF-Bericht über den Well-Architected-Status von FSx for ONTAP-Dateisystemen

Nach der Analyse können Sie einen PDF-Bericht über den Well-Architected-Status Ihrer FSx for ONTAP-Dateisysteme für Ihre Unterlagen herunterladen oder mit Ihren Teams teilen. link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Überprüfen Sie den Status Ihrer FSx for ONTAP-Dateisysteme hinsichtlich ihrer Architektur in NetApp Workload Factory."]



=== Datenbank-Workloads



==== Gut konzipierte Analyse für Oracle

Die gut durchdachte Analyse umfasst die folgenden neuen Konfigurationen für Oracle:

* *Backup-Konfiguration*: Prüft, ob Ihre Oracle-Datenbanken durch AWS Backup oder FSx for ONTAP Backups geschützt sind.
* *Klonbereinigung*: Es wird empfohlen, veraltete oder ungenutzte Klone zu entfernen, um Speicherkosten zu reduzieren.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementieren Sie gut strukturierte Datenbankkonfigurationen in Workload Factory"]



==== Automatische Kosteneinsparungsberechnung für Oracle-Datenbanken auf Amazon Elastic Block Store (EBS)

Workload Factory analysiert automatisch die Kosten des für Oracle-Datenbanken verwendeten EBS-Speichers und schlägt anschließend die optimale und kosteneffizienteste FSx for ONTAP Konfiguration vor. Sie können mehrere EBS-Hosts zur Analyse auswählen und dann die Kosten für den Betrieb dieser Hosts auf EBS im Vergleich zu FSx for ONTAP vergleichen.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Entdecken Sie die Einsparungen für erkannte EBS-Hosts"]



==== CRR-Konfigurationsvoraussetzungsprüfung

Wenn Sie die regionsübergreifende Replikation (CRR) für Ihren Datenbankbestand aktivieren, prüft die Database-Workload, ob das FSx for ONTAP-Dateisystem über eine zugehörige Verbindung und Replikationseinrichtung verfügt. Falls erforderlich, leitet die Database-Workload Sie zum FSx for ONTAP Storage-Workload-Replikationsassistenten weiter, um die Verbindungsvoraussetzungen auf die gleiche Weise wie bei Storage zu erfüllen.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementieren Sie gut strukturierte Datenbankkonfigurationen in Workload Factory"]



==== Zugehörige LUNs für Microsoft SQL Server-Datenbanken

Sie können jetzt im Inventarbildschirm von Workload Factory die Microsoft SQL Server-Datenbanken zugeordneten LUNs anzeigen. So erkennen Sie schnell, welche LUNs mit welchen Datenbanken verbunden sind, was die Verwaltung und Optimierung Ihrer Speicherressourcen erleichtert.



=== Einrichtung und Administration



==== E-Mail-Benachrichtigungen für Workload Factory

Sie können Workload Factory-Benachrichtigungen jetzt per E-Mail erhalten, ohne ein Amazon SNS-Thema konfigurieren zu müssen. Aktivieren Sie E-Mail-Benachrichtigungen, um Benachrichtigungen an alle mit dem Konto verknüpften E-Mail-Adressen zu senden.

Sie können E-Mail-Benachrichtigungen entweder über die Workload Factory Console oder die NetApp Console konfigurieren. Die NetApp Console bietet außerdem die Möglichkeit, NetApp Console-Benachrichtigungen zu konfigurieren, die unter dem Glockensymbol angezeigt werden.

Wenn Sie Benachrichtigungen in der NetApp Console konfigurieren, übernimmt die NetApp Console die Zustellung der Benachrichtigungen. Die Benachrichtigungseinstellungen in der NetApp Workload Factory-Konsole werden schreibgeschützt, um doppelte Benachrichtigungen zu vermeiden.



==== Neue Benachrichtigung für EDA-Latenz

Der NetApp Workload Factory-Benachrichtigungsdienst umfasst die Benachrichtigung für EDA-Latenz, die Sie benachrichtigt, wenn die Latenz einer EDA-Operation einen bestimmten Schwellenwert überschreitet.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Workload Factory-Benachrichtigungen konfigurieren"]



==== Neue Benachrichtigung für FSx für ONTAP-Dateisysteme

Der NetApp Workload Factory-Benachrichtigungsdienst beinhaltet eine Benachrichtigung mit einer monatlichen Kostenübersicht für Ihr Konto, die Ausgabentrends, ungewöhnliche Änderungen und die besten Einsparmöglichkeiten für FSx für ONTAP-Dateisysteme aufzeigt.

link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["Workload Factory-Benachrichtigungen konfigurieren"]



=== EDA-Workloads



==== E-Mail- und SNS-Benachrichtigungen für Latenzereignisse

Sie können jetzt automatische Benachrichtigungen erhalten, wenn Latenzereignisse auf Ihren FSx for ONTAP Volumes erkannt werden. Konfigurieren Sie E-Mail- oder Amazon SNS-Benachrichtigungen in den Workload Factory-Einstellungen, um Echtzeit-Benachrichtigungen zu erhalten, wenn Volumes Ihre konfigurierten Latenzschwellenwerte überschreiten.

Die Benachrichtigungen werden auf Dateisystembasis versendet und enthalten Details zu allen betroffenen Volumes, Schweregraden, Schwellenwertvergleichen und einen direkten Link zur Latenzseite für Untersuchungen.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["Erfahren Sie, wie Sie Latenzbenachrichtigungen konfigurieren"^].



==== Latenztrendvisualisierung mit interaktiven Diagrammen

Das Panel zur Latenzanalyse verfügt nun über eine Registerkartenoberfläche für eine umfassende Latenzanalyse:

* *Übersicht*: Sehen Sie sich die automatisierten Ergebnisse der Basisanalyse an, die zeigen, welche Komponente die Latenz verursacht
* *Im Zeitverlauf*: Visualisieren Sie das Volumen-Latenzverhalten mit einem interaktiven Diagramm über mehrere Zeiträume (1 Stunde, 3 Stunden, 12 Stunden, 24 Stunden oder 72 Stunden)


Die Registerkarte *Über die Zeit* zeigt Rohdaten zur CloudWatch-Latenz (Lesen oder Schreiben, je nachdem, welcher Alarm ausgelöst wurde) mit Warn- und kritischen Schwellenwertlinien, Indikatoren für Überschreitungen und detaillierten Metriken für jede Verletzung an.

Diese umfassende Visualisierung hilft Ihnen, Muster zu erkennen, zu verstehen, ob Latenzprobleme wiederkehrend oder vereinzelt auftreten, und fundiertere Entscheidungen über die Behebung zu treffen.

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["Lernen Sie, wie Sie Latenztrends analysieren"^].

==== Gut strukturierte Analyse-Updates

Workload Factory analysiert Ihre FSx for ONTAP-Dateisysteme auf *inaktive NAS-Volumes*. Die Konfigurationsanalyse identifiziert nicht aktiv genutzte NAS-Volumes und empfiehlt Maßnahmen zur Optimierung der Speichernutzung und Kostensenkung.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Implementieren Sie gut strukturierte FSx for ONTAP Dateisysteme"]

Die Journaltabellenfunktion bietet eine Möglichkeit, den FSx ONTAP S3-Objektspeicher in großem Maßstab zu verwalten und ermöglicht es Ihnen, Benutzerzugriffsereignisse und Operationen an Objekten in Ihren Buckets einfach zu protokollieren. Mit Workload Factory können Sie automatische Scans Ihrer Buckets und S3-Tabellen einrichten, um eine Aufzeichnung der Ereignisse zu führen, die alle Objekte, deren Details und Benutzeraktionen anzeigt.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["Richten Sie die Journaltabelleninfrastruktur für NetApp Workload Factory ein."]



==== Unterstützung für die Verwaltung bestehender Initiatorgruppen

Workload Factory unterstützt die Verwaltung bestehender Initiatorgruppen. Sie können Blockgeräte und Host-Initiatoren hinzufügen oder entfernen, um Ihre bereitgestellte Workload termingerecht auszuliefern.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Blockspeicher in NetApp Workload Factory verwalten"]



==== Unterstützung für die Angabe des Aggregats für FlexVol volumes

Workload Factory unterstützt das Erstellen von FlexVol Volumes auf einem ausgewählten Aggregat. Dadurch können Sie die Kapazitäts- und Leistungslast auf alle Dateisystemressourcen verteilen.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["Erstellen Sie ein FlexVol Volume in NetApp Workload Factory"]



==== Letzter Zugriffszeitpunkt für Blockgeräte

Workload Factory meldet den letzten Zugriffszeitpunkt für Blockgeräte. Dieser Zeitpunkt basiert auf dem letzten Zugriff auf die Initiatorgruppe in Workload Factory. Die Kenntnis des letzten Zugriffszeitpunkts jedes Blockgeräts ermöglicht die Optimierung Ihrer Ressourcen.



=== Datenbank-Workloads



==== Gut konzipierte Analyse für Oracle

Die gut durchdachte Analyse umfasst die folgenden neuen Konfigurationen für Oracle:

* *Oracle Critical Patch Updates*: Bewertet fehlende Sicherheitspatches für unterstützte selbstverwaltete Oracle-Datenbankinstallationen.
* *Regionsübergreifende Replikation*: Prüft, ob die regionsübergreifende Replikation (CRR) für Ihre FSx for ONTAP Dateisysteme, die Oracle Datenbanken bedienen, aktiviert ist.
* *Applikationskonsistente Snapshots*: prüft, ob die Oracle Datenbank durch applikationskonsistente Snapshots mit NetApp SnapCenter geschützt ist.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementieren Sie gut strukturierte Datenbankkonfigurationen in Workload Factory"]



==== Einmalige Bewertung für Oracle

Workload Factory bietet jetzt eine einmalige Analyse Ihrer Oracle Datenbank-Bereitstellungen – wann immer Sie diese benötigen: bei Neuinstallationen, nach Migrationen, bei der Fehlerbehebung und mehr. Sie erhalten Empfehlungen zur Optimierung Ihrer Bereitstellungskonfigurationen und zur Einhaltung von Best Practices. Dies ist Ihre Gelegenheit, die well-architected Funktion zu nutzen, ohne sich vollständig auf den Einrichtungsprozess festlegen zu müssen. Sie können Ihre Datenbanken analysieren, ohne Anmeldeinformationen in Workload Factory zu speichern, Datenbankressourcen zu registrieren oder externe Verbindungen einzurichten.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Führen Sie eine einmalige Bewertung Ihrer Oracle-Datenbankbereitstellungen durch"]



==== Verbesserte Kosteneinsparungen für mehrere Oracle-Datenbanken

Wir haben die Kostenanalyse für Oracle optimiert, um eine konsolidierte Speicherlösung für mehrere lokale Oracle-Datenbanken auf einem FSx for ONTAP-Dateisystem bereitzustellen. Diese Konsolidierung optimiert die Speicherkosten, indem die Anzahl der für mehrere Datenbanken benötigten Dateisysteme reduziert wird, was zu höheren Kosteneinsparungen führt.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Entdecken Sie Einsparungen für erkannte Datenbanken auf Oracle On-Premises."]

=== EDA-Workloads



==== EDA-Kapazitätsempfehlungen

Workload Factory analysiert nun automatisch die SSD-Kapazität jedes FSx for ONTAP-Clusters in Ihrem EDA-Inventar und generiert Kapazitätsempfehlungen, wenn Anpassungen erforderlich sind. Der Algorithmus wird einmal alle 24 Stunden für jeden Cluster ausgeführt.

Wenn eine Empfehlung identifiziert wird:

* Sie erhalten umgehend eine E-Mail oder eine WAD-Benachrichtigung (nicht Teil der wöchentlichen Zusammenfassung).
* In der Tabelle „Cluster“ wird für den betroffenen Cluster eine Glühbirnenanzeige eingeblendet.
* Eine Zusammenfassung im Übersichts-Dashboard zeigt die Gesamtzahl der Cluster mit aktiven Empfehlungen an.


Wählen Sie einen Cluster mit einer aktiven Empfehlung aus, um die Empfehlung anzuzeigen, in der die vorgeschlagene Änderung und die Begründung erläutert werden.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["Erfahren Sie mehr über Kapazitätsempfehlungen"^].



==== Automatisierte KI-Latenzanalyse

Die Latenzanalyse umfasst nun auch die Analyse von KI-Agenten.

Bei komplexen Szenarien mit Daten- oder Clusterlatenz können Sie eine KI-Agentenanalyse aufrufen, um Folgendes zu erhalten:

* Detaillierte Ursachenanalyse
* Liste der betroffenen EC2-Clients
* Spezifische Abhilfemaßnahmen


Die Analyse mittels KI-Agent erfordert die Konfiguration des Amazon Bedrock-Modell-ARN und ist bedarfsgesteuert verfügbar, wenn eine Basisanalyse eine weiterführende Untersuchung empfiehlt. Der KI-Agent kann Probleme wie übermäßige Speicherauslastung, suboptimale Mount-Point-Konfigurationen, FlexGroup-Rebalancing-Bedarf und Skalierungsanforderungen identifizieren.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["Erfahren Sie mehr über die Analyse von KI-Agenten"^].



== 09. März 2026



=== VMware-Workloads



==== Einführung der Unterstützung für Migration Advisor für Google Cloud VMware Engine

NetApp Workload Factory unterstützt jetzt Google Cloud VMware Engine. Sie können Ihre lokalen VMware-Workloads zu Google Cloud VMware Engine migrieren, indem Sie Google Cloud NetApp Volumes als externe NFS-Datenspeicher verwenden. Dadurch werden die Kosten optimiert und Sie erhalten mehr Kontrolle über Ihre VMware-Umgebung, ohne Ihre Anwendungen refaktorisieren oder auf eine neue Plattform umstellen zu müssen.

Der Google Cloud VMware Engine Migrationsberater bietet:

* Automatisierte Dimensionierung und Auslegung von Google Cloud VMware Engine-Clustern mit Unterstützung für VE1- und VE2-Knotentypen
* VM-Layoutoptimierung zwischen internen vSAN-Datenspeichern und Google Cloud NetApp Volumes externen NFS-Datenspeichern.
* Detaillierter Kostenvergleich und TCO-Analyse zwischen reinen vSAN- und Google Cloud NetApp Volumes-basierten Konfigurationen.
* Unterstützung für die Servicelevel Standard, Premium und Extreme des Google Cloud NetApp Volumes.
* Infrastructure-as-Code-Bereitstellung mit Terraform für die automatisierte Bereitstellung.
